    • This program is a database management system for the Academy Awards. Two files, one for movies and one for actors, are read from a file into separate vectors. Then, the user may search, modify, sort, display, and save the records. Currently, all functions work in this program and compile successfully on csegrid.

    Menu Functions

    The user interface scheme is a switch based system. Each menu the user is shown lists each choice with an integer in front, with the final option always being to return to the previous menu or quit (in the case of the main menu).

    An example of the main menu and first submenu is shown to the left. When the user selects ‘1’ or ‘5’ they are shown a message that the action as been successful and are shown no further sub-menus. In this case, the user selects '2' and is adding a movie record.

    Here is a screenshot of a user selecting '4' for the sort function. Once the user then selects one of those choices, the search menu will prompt the user to enter their search criteria, and the sort menu will begin displaying the sorted vector.

    These submenus continue to loop if the user wishes to continue the selected action. For example, if the user sorted movies by year, the program will continue to prompt the user to sort for another field until the user selects to navigate back to the previous menu.

    The submenus are error tolerant and will continue to loop if the user enters a character or an integer that is out of bounds for the menu.

    Key Algorithms

    One key algorithm is the insertion sort algorithm which is used to sort each attribute for both classes. This algorithm iterates through the vector and compares the current item’s attribute to the next item’s attribute. If the next item’s attribute is smaller than the current item’s, then that item is swapped successively to the left until it is larger than the item to its left. The time complexity for the insertion sort is O(n^2).

    Another key algorithm is the partial/complete search algorithm. This algorithm iterates through the vector from beginning to end checking each object’s member variables against the search terms that the user input. This is done with a for loop and an if statement inside. The time complexity for this algorithm is O(n)
